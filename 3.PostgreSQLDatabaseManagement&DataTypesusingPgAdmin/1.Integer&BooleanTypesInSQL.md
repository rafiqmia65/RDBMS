# Integer & Boolean Types in SQL & PostgreSQL

This document provides a overview of **Integer** and **Boolean** data types in **SQL**, with a specific focus on **PostgreSQL**. It is intended for database learners, backend developers, and anyone designing relational databases.

---

## 1. Integer Data Types

Integer data types are used to store **whole numbers** (positive, negative, or zero) without fractional parts. They are commonly used for IDs, counters, quantities, and numeric flags.

### 1.1 Common Integer Types in SQL

| Type     | Description         |
| -------- | ------------------- |
| SMALLINT | Small-range integer |
| INTEGER  | Standard integer    |
| BIGINT   | Large-range integer |

> Note: Exact size and range may vary slightly between database systems.

---

### 1.2 Integer Types in PostgreSQL

PostgreSQL provides multiple integer types with different storage sizes and ranges.

| Type     | Size    | Range                                                   |
| -------- | ------- | ------------------------------------------------------- |
| SMALLINT | 2 bytes | -32,768 to 32,767                                       |
| INTEGER  | 4 bytes | -2,147,483,648 to 2,147,483,647                         |
| BIGINT   | 8 bytes | -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 |

---

### 1.3 Choosing the Right Integer Type

- Use **SMALLINT** for small values (e.g., age, rating, status codes)
- Use **INTEGER** for most general-purpose numeric values
- Use **BIGINT** for large counters, financial data, or systems with high-scale growth

Choosing the correct type helps optimize **storage**, **performance**, and **scalability**.

---

### 1.4 Example: Integer Usage

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  age INTEGER,
  login_count BIGINT
);
```

---

## 2. Boolean Data Types

Boolean data types are used to represent **true/false** values. They are commonly used for flags, statuses, and conditions.

---

### 2.1 Boolean Type in SQL

Standard SQL defines a **BOOLEAN** type, but its implementation may differ across databases.

Typical logical values:

- TRUE
- FALSE
- NULL (unknown or not set)

---

### 2.2 Boolean Type in PostgreSQL

PostgreSQL has native support for the **BOOLEAN** type.

| Value | Accepted Inputs              |
| ----- | ---------------------------- |
| TRUE  | TRUE, 't', 'true', '1', yes  |
| FALSE | FALSE, 'f', 'false', '0', no |

Internally, PostgreSQL stores BOOLEAN values efficiently using **1 byte**.

---

### 2.3 Example: Boolean Usage

```sql
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  is_paid BOOLEAN DEFAULT false,
  is_active BOOLEAN NOT NULL
);
```

---

### 2.4 Boolean with Conditions

```sql
SELECT *
FROM orders
WHERE is_paid = true;
```

You can also write:

```sql
SELECT *
FROM orders
WHERE is_paid;
```

---

## 3. Integer vs Boolean: When to Use What

| Use Case                | Recommended Type |
| ----------------------- | ---------------- |
| Numeric calculations    | INTEGER / BIGINT |
| Counters                | INTEGER / BIGINT |
| Yes/No flags            | BOOLEAN          |
| Status enabled/disabled | BOOLEAN          |

> Avoid using integers like `0` and `1` to represent boolean logic when BOOLEAN is available.

---

## 4. Best Practices

- Always choose the **smallest integer type** that safely fits your data
- Use **BOOLEAN** instead of numeric flags for clarity
- Apply **DEFAULT** values where appropriate
- Use **NOT NULL** constraints for required fields
